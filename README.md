# Анализ доходности и волатильности криптовалют

Итоговый проект по дисциплине **«Анализ данных на Python»** в НИУ ВШЭ — Пермь.

**Авторы:** Степан Козлов, Александр Шембек  
**Источник данных:** CoinGecko API  
**Объекты анализа:** BTC, ETH, BNB, SOL, XRP, DOGE, ADA и TRX

[Открыть основной ноутбук](./crypto_analysis.ipynb)

## Цель проекта

Сравнить доходность и риск восьми популярных криптовалют за 365 дней, изучить взаимосвязи между их дневными доходностями и проверить четыре исследовательские гипотезы.

## Исследовательские гипотезы

1. BTC и ETH менее волатильны, чем большинство альткоинов.
2. Более высокая доходность связана с более высоким риском.
3. Дневные доходности криптовалют положительно коррелируют между собой.
4. DOGE более волатилен, чем крупные инфраструктурные криптовалюты.

Гипотезы → Сбор данных → Очистка → Создание признаков → Анализ → Визуализация → Выводы

## Сбор данных

Для каждой криптовалюты через endpoint `market_chart` загружаются ежедневные значения:

- цены в долларах США;
- рыночной капитализации;
- общего объёма торгов;
- временной метки наблюдения.

В контрольном запуске использовался период **с 15 июня 2025 года по 14 июня 2026 года**.

Исходная выборка содержала **2928 строк**. После удаления повторных комбинаций `symbol + date` осталось **2920 наблюдений** — по 365 дней для каждой из восьми криптовалют.

При повторном запуске через API период и результаты могут измениться, поскольку CoinGecko возвращает данные за последние 365 дней относительно даты запуска.

## Рассчитанные показатели

### Дневная доходность

```text
R_t = P_t / P_(t-1) - 1
```

Показывает относительное изменение цены по сравнению с предыдущим днём.

### Накопленная доходность

```text
CR_t = P_t / P_0 - 1
```

Показывает изменение цены относительно первого дня рассматриваемого периода.

### Годовая волатильность

```text
σ_year = σ_day × √365
```

Стандартное отклонение дневных доходностей переводится в годовой масштаб. Используется 365 дней, поскольку криптовалютный рынок работает ежедневно.

Также рассчитаны:

- 30-дневная скользящая волатильность;
- максимальная просадка;
- средняя и медианная дневная доходность;
- упрощённое отношение накопленной доходности к волатильности;
- корреляционная матрица дневных доходностей.

## Методы анализа

- группировки и агрегирование данных;
- расчёт средних, медиан и стандартного отклонения;

## Основные результаты контрольного запуска

| Криптовалюта | Накопленная доходность | Годовая волатильность | Максимальная просадка |
|---|---:|---:|---:|
| TRX | 17,06% | 28,50% | −26,49% |
| BNB | −5,29% | 53,58% | −56,40% |
| ETH | −33,84% | 67,89% | −67,52% |
| BTC | −38,81% | 43,00% | −51,22% |
| XRP | −46,41% | 67,98% | −69,26% |
| DOGE | −51,05% | 79,44% | −71,90% |
| SOL | −52,78% | 72,03% | −74,89% |
| ADA | −72,54% | 77,81% | −83,73% |

Ключевые выводы для выбранного периода:

- TRX оказался единственной криптовалютой с положительной накопленной доходностью;
- TRX одновременно показал минимальную годовую волатильность;
- DOGE оказался самым волатильным активом;
- ADA показала худшую накопленную доходность и максимальную просадку;
- высокая волатильность не обеспечила более высокую доходность;
- дневные доходности выбранных криптовалют в основном положительно коррелировали;
- TRX был слабее связан с остальной частью рынка.

Эти выводы относятся только к рассматриваемой выборке и не доказывают, что аналогичные закономерности сохранятся в другие периоды.

## Запуск проекта

### 1. Клонирование

```bash
git clone https://github.com/Stepanda1/Analyze-profitability-and-volatility-crypto.git
cd Analyze-profitability-and-volatility-crypto
```

### 2. Виртуальное окружение

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

Linux/macOS:

```bash
source .venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка CoinGecko API

Создайте файл `.env` по образцу `.env.example`:

```text
COINGECKO_API_KEY=your_api_key_here
```

### 5. Запуск ноутбука

```bash
jupyter notebook crypto_analysis.ipynb
```

Без API-ключа ноутбук может использовать локальный файл `raw_crypto_data.csv`, расположенный рядом с ним.

Обязательные столбцы локального файла:

```text
date,symbol,coin_id,price,market_cap,total_volume
```

## Ограничения исследования

- Рассматривается только один год, поэтому результаты нельзя автоматически переносить на другие периоды.
- Работа описывает исторические данные и не прогнозирует будущие цены.
- Гипотезы оцениваются преимущественно с помощью описательных показателей и визуализаций, без формальных тестов статистической значимости.
- Показатель `return_to_risk` является упрощённым и не заменяет коэффициент Шарпа.
- Данные последнего дня могут быть неполными, если запрос выполнен до завершения суток.
- Результаты не являются инвестиционной рекомендацией.
